# Final release hardening

Phase 6.7 is the final release-hardening layer for RhoDyn. It verifies that the
software package can be prepared for PyPI-style distribution, that the public
GitHub and Zenodo release surfaces resolve without authentication, that
documentation links resolve locally, and that declared dependencies have been
reviewed.

The four release-hardening reports are generated by these commands.

```bash
python scripts/pypi_dry_run.py
python scripts/zenodo_dry_run.py
python scripts/check_public_release_integrity.py
python scripts/check_docs_links.py
python scripts/check_dependency_security.py
python scripts/audit_stage4_docker_smoke.py --out docs/docker_smoke_audit_report.json
python scripts/run_stage5_screenshot_regression.py
```

The PyPI and Zenodo dry runs do not upload, reserve, publish, or mutate remote
records. They validate local packaging and metadata only. The public-release
integrity check is the live network check for the already published GitHub and
Zenodo surfaces. The documentation link scan checks local links and URL syntax
by default, while the dependency review is a deterministic offline review of
declared Python optional dependencies and the Node lockfile. A live advisory
database scan can still be run before a future PyPI upload if network policy and
credentials allow it.

The release remains scientifically separate from the RhoA/microglia manuscript.
RhoDyn can reproduce public examples and optional case studies, but the
manuscript was not generated with RhoDyn.
